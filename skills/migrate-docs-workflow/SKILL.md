# Migrate Docs Workflow

## Use specs for design

- Keep design documents in `specs/`.
- Record enough detail to derive implementation with minimal extra decisions.

## Use PROGRESS.md for execution status

- Track completed work, current work, and next steps in `PROGRESS.md`.
- Update `PROGRESS.md` when finishing major milestones.
- Update `PROGRESS.md` during active development when plan status changes.
